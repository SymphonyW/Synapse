# 部署与启动

本文档说明如何从零启动 Synapse。当前仓库提供 Docker Compose 本地编排，不包含 k8s、Nginx 或正式生产部署配置。

| 你现在想做什么 | 直接看这里 |
|---|---|
| 先在 5 分钟内跑起来 | [Compose 启动](#compose-启动) |
| 先看完整链路，不想配 Key | 保持默认 `mock` provider |
| 接真实模型 | [Provider 启动模式](#provider-启动模式) |
| 做前端开发 | [Compose 启动](#compose-启动) 或 [本地分组件启动](#本地分组件启动) |

## 前置依赖

| 依赖 | 用途 | 说明 |
|---|---|---|
| Docker Desktop / Docker Compose v2 | 启动 gateway、ai-engine、postgres、redis | 推荐 |
| PowerShell | 执行 [scripts/dev.ps1](../scripts/dev.ps1) | Windows 推荐 |
| Go | 本地运行 Gateway 和测试 | `go.mod` 声明 1.25.0 |
| Python | 本地运行 AI Engine 和测试 | Dockerfile 使用 3.12 |
| Node.js + npm | 本地运行 Web 或构建静态产物 | package 未声明 engines，版本待确认，建议当前 LTS |
| protoc | 重新生成 proto 代码 | 仅修改 proto 时需要 |

## Compose 启动

[docker-compose.yml](../docker-compose.yml) 默认启动：

| 服务 | 端口 | 说明 |
|---|---|---|
| gateway | 8080 | HTTP API |
| ai-engine | 50051 | gRPC |
| postgres | 15432 | 宿主机端口；容器内仍是 5432 |
| redis | 6379 | 队列 |
| web | 5173 | 生产静态 Web，Nginx 反代 `/v1` 与 `/healthz` 到 gateway |

```powershell
.\scripts\dev.ps1 -Task up
```

这就是默认推荐路径：`mock` provider、Docker Compose 一键启动、Web 也随服务一起拉起。

如果需要本地 Vite 开发模式，再单独启动 Web：

```powershell
Set-Location apps/web
npm install
npm run dev
```

如需容器内 Vite 开发模式：

```powershell
docker compose --profile web-dev up --build web-dev
```

停止 Compose：

```powershell
.\scripts\dev.ps1 -Task down
```

## Provider 启动模式

默认 `SYNAPSE_MODEL_PROVIDER=mock`，不需要 API Key。

| 模式 | 准备文件 | 启动命令 |
|---|---|---|
| OpenAI | `Copy-Item docker-compose.openai.env.example docker-compose.openai.env` | `.\scripts\dev.ps1 -Task up-openai` |
| Gemini | `Copy-Item docker-compose.gemini.env.example docker-compose.gemini.env` | `.\scripts\dev.ps1 -Task up-gemini` |
| 智谱 | `Copy-Item docker-compose.zhipu.env.example docker-compose.zhipu.env` | `.\scripts\dev.ps1 -Task up-zhipu` |
| 镜像源 | `Copy-Item docker-compose.mirror.env.example docker-compose.mirror.env` | `.\scripts\dev.ps1 -Task up-mirror` |
| 向量记忆 | `Copy-Item docker-compose.vector.env.example docker-compose.vector.env` | `docker compose --env-file docker-compose.vector.env up --build -d` |
| 智谱 + 镜像源 | 准备 `docker-compose.zhipu.env` 和 `docker-compose.mirror.env` | `.\scripts\dev.ps1 -Task up-zhipu-mirror` |

provider env 文件需要填写真实 API Key。不要提交非 example env 文件。

真实模型长文本建议：

| 配置 | 建议 |
|---|---|
| `SYNAPSE_OPENAI_MAX_TOKENS` | 按 provider 能力调大单次输出上限 |
| `SYNAPSE_OPENAI_CONTINUATION_MAX_ROUNDS` | 长文、报告类任务建议保留默认 `8` 或更高 |
| `SYNAPSE_OPENAI_LONG_FORM_MIN_CHARS` | 控制长文本接受完成前的最低字符预算，默认 `2400` |
| `SYNAPSE_TASK_EXEC_TIMEOUT` | Compose 默认 `300s`，避免 Worker 过早取消长任务 |
| `SYNAPSE_HTTP_WRITE_TIMEOUT` | Compose 默认 `300s`，避免 SSE 长输出被 HTTP 写超时截断 |

## 本地分组件启动

安装依赖：

```powershell
Set-Location apps/web
npm install
Set-Location ..\..

python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -U pip
python -m pip install -r services/ai-engine-py/requirements.txt
```

启动 Postgres/Redis：

```powershell
docker compose up -d postgres redis
```

设置 Gateway 连接本地依赖：

```powershell
$env:SYNAPSE_DATABASE_URL = "postgres://synapse:synapse@127.0.0.1:15432/synapse?sslmode=disable"
$env:SYNAPSE_REDIS_ADDR = "127.0.0.1:6379"
$env:SYNAPSE_AI_ENGINE_ADDR = "127.0.0.1:50051"
```

如修改过 proto，生成代码：

```powershell
.\scripts\dev.ps1 -Task proto
```

启动三个终端：

```powershell
# 终端 A
.\scripts\dev.ps1 -Task ai

# 终端 B
.\scripts\dev.ps1 -Task gateway

# 终端 C
.\scripts\dev.ps1 -Task web
```

## 数据库初始化

无需手动执行 SQL。Gateway 连接 Postgres 成功后，会在启动时自动创建：

1. `tasks`
2. `task_events`
3. `dead_letter_tasks`
4. `auth_users`
5. `auth_sessions`

若启用 `SYNAPSE_MEMORY_BACKEND=vector`，AI Engine 还会在启动时确保 `pgvector` 扩展、`vector_memories` 表和相关索引存在。Compose 默认使用自带 pgvector 的 PostgreSQL 镜像；若向量后端配置不完整或数据库初始化失败，AI Engine 会直接启动失败。

如果 Postgres 不可用，Gateway 会回退内存存储。该模式仅适合开发调试，重启后数据丢失。

## 验证命令

```powershell
$base = "http://127.0.0.1:8080"
curl.exe -s "$base/healthz"
```

登录并创建任务：

```powershell
curl.exe -s -c .\synapse.cookies `
  -H "Content-Type: application/json" `
  -d '{"username":"admin","password":"123456"}' `
  "$base/v1/auth/login"

$response = curl.exe -s -b .\synapse.cookies `
  -H "Content-Type: application/json" `
  -d '{"prompt":"hello synapse","metadata":{"client_view":"chat"}}' `
  "$base/v1/tasks"

$task = $response | ConvertFrom-Json
curl.exe -N -b .\synapse.cookies "$base/v1/tasks/$($task.id)/events"
```

更完整的验证步骤见 [05-接口验证手册](05-接口验证手册.md)。

## 脚本任务

| 任务 | 说明 |
|---|---|
| `proto` | 同时生成 Go/Python proto 代码 |
| `proto-go` | 只生成 Go proto 代码 |
| `proto-py` | 只生成 Python proto 代码 |
| `gateway` | 本地运行 Gateway |
| `ai` | 本地运行 AI Engine |
| `web` | 本地运行 Web |
| `agent-regression` | 运行 Agent 回归评测 |
| `verify-agent-mode` | 打印当前 Agent 关键配置 |
| `up` | Compose mock 模式后台启动 |
| `up-mirror` | 使用镜像源配置启动 |
| `up-openai` / `up-gemini` / `up-zhipu` | 使用对应 provider env 启动 |
| `up-*-mirror` | provider env 与 mirror env 组合启动 |
| `down` | 停止 Compose |
