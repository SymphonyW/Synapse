# 部署与启动

## 1. 目标

本文档说明 Synapse 在开发环境的标准启动方式、常见组合命令与环境文件组织方式。

## 2. 启动入口

统一脚本入口位于 [scripts/dev.ps1](../scripts/dev.ps1)。

支持任务：

1. proto / proto-go / proto-py
2. gateway / ai / web
3. up / up-openai / up-gemini / up-zhipu / up-zhipu-mirror
4. down

## 3. Docker Compose 启动

编排文件位于 [docker-compose.yml](../docker-compose.yml)，默认启动组件：

1. gateway
2. ai-engine
3. postgres
4. redis

常用命令：

```powershell
.\scripts\dev.ps1 -Task up
.\scripts\dev.ps1 -Task down
```

## 4. Provider 启动模式

环境模板：

1. [docker-compose.openai.env.example](../docker-compose.openai.env.example)
2. [docker-compose.gemini.env.example](../docker-compose.gemini.env.example)
3. [docker-compose.zhipu.env.example](../docker-compose.zhipu.env.example)
4. [docker-compose.mirror.env.example](../docker-compose.mirror.env.example)

对应启动命令：

```powershell
.\scripts\dev.ps1 -Task up-openai
.\scripts\dev.ps1 -Task up-gemini
.\scripts\dev.ps1 -Task up-zhipu
.\scripts\dev.ps1 -Task up-zhipu-mirror
```

## 5. 本地分组件启动

1. 生成协议代码：

```powershell
.\scripts\dev.ps1 -Task proto
```

2. 分别启动：

```powershell
.\scripts\dev.ps1 -Task ai
.\scripts\dev.ps1 -Task gateway
.\scripts\dev.ps1 -Task web
```

## 6. 端口约定

1. Web: 5173
2. Gateway HTTP: 8080
3. AI Engine gRPC: 50051
4. PostgreSQL: 5432
5. Redis: 6379

## 7. 常见问题

1. 模型不返回 token：优先检查 provider 相关 env 文件与 base_url 是否匹配。
2. docker 拉取慢：使用 mirror env 启动模式。
3. 本地没 Postgres/Redis：Gateway 会自动回退内存实现，但数据不持久。
