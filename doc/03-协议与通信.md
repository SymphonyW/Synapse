# 协议与通信

## 1. 协议文件

主协议定义在 [proto/synapse/v1/agent.proto](../proto/synapse/v1/agent.proto)。

该文件定义了 Gateway 与 AI Engine 的 gRPC 契约，并作为跨语言代码生成的唯一来源。

## 2. RPC 服务

1. Health(HealthRequest) returns HealthResponse
2. SubmitTask(SubmitTaskRequest) returns stream AgentEvent

## 3. 关键消息

1. SubmitTaskRequest
2. task_id：任务主键
3. user_id：归属用户
4. prompt：提交给模型的输入
5. metadata：补充字段（如 conversation_id, model_messages_json）

6. AgentEvent
7. type：事件类型枚举
8. message：通用信息
9. token：增量文本
10. trace_id：链路追踪标识
11. emitted_at_unix_ms：事件产生时间

## 4. 双协议链路

1. 内部通信：Gateway <-> AI Engine 使用 gRPC Streaming。
2. 对外通信：Gateway -> Browser 使用 HTTP + SSE。

该设计让内部服务可保持强类型协议，同时前端使用浏览器原生 EventSource 即可消费流式结果。

## 5. 代码生成

1. Go 生成目标目录： [services/gateway-go/internal/gen](../services/gateway-go/internal/gen)
2. Python 生成目标目录： [services/ai-engine-py/synapse](../services/ai-engine-py/synapse)
3. Python 生成后通过 [scripts/post_gen.py](../scripts/post_gen.py) 补齐 __init__.py

## 6. 事件语义映射

AI Engine 输出枚举事件，Gateway 在 Worker 中将其归一化为小写字符串，用于 SSE 输出与持久化。

示例：

1. AGENT_EVENT_TYPE_STARTED -> started
2. AGENT_EVENT_TYPE_TOKEN -> token
3. AGENT_EVENT_TYPE_COMPLETED -> completed
4. AGENT_EVENT_TYPE_FAILED -> failed

## 7. 兼容性建议

1. 新增字段优先走 proto 可选字段或 map 元数据，保持向后兼容。
2. 事件类型新增后，需同步更新前端事件监听集合。
3. 任何 break change 都应先升级生成代码并补充回归测试。
