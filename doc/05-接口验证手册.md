# 接口验证手册

本文档提供一组可复制的 PowerShell 命令，用来验证 Synapse 从健康检查、认证、任务、SSE 到长期记忆的基础流程。

## 前提

启动后端：

```powershell
.\scripts\dev.ps1 -Task up
```

如果需要 Web：

```powershell
Set-Location apps/web
npm install
npm run dev
```

设置 API 地址：

```powershell
$base = "http://127.0.0.1:8080"
```

## 健康检查

```powershell
curl.exe -s "$base/healthz"
```

mock 模式预期：

```json
{"status":"ok","ai_engine":"ok","model_provider":"mock"}
```

如果返回 `503` 或 `degraded`，优先检查 `ai-engine` 容器或本地 AI Engine 是否运行。

## 登录管理员

默认管理员由 Gateway 启动时 upsert：

| 用户名 | 密码 |
|---|---|
| `admin` | `123456` |

```powershell
curl.exe -s -c .\synapse.cookies `
  -H "Content-Type: application/json" `
  -d '{"username":"admin","password":"123456"}' `
  "$base/v1/auth/login"
```

查询当前用户：

```powershell
curl.exe -s -b .\synapse.cookies "$base/v1/auth/me"
```

## 注册普通用户

```powershell
curl.exe -s `
  -H "Content-Type: application/json" `
  -d '{"username":"devuser","password":"123456"}' `
  "$base/v1/auth/register"
```

如果用户已存在，会返回 `409`。可继续用该用户登录：

```powershell
curl.exe -s -c .\synapse-user.cookies `
  -H "Content-Type: application/json" `
  -d '{"username":"devuser","password":"123456"}' `
  "$base/v1/auth/login"
```

## 创建任务并查看事件

```powershell
$response = curl.exe -s -b .\synapse.cookies `
  -H "Content-Type: application/json" `
  -d '{"prompt":"hello synapse, calculate 8 * 9 and summarize it","metadata":{"client_view":"chat","agent_enabled":"true","memory_write_enabled":"true"}}' `
  "$base/v1/tasks"

$task = $response | ConvertFrom-Json
$task.id
```

查询任务：

```powershell
curl.exe -s -b .\synapse.cookies "$base/v1/tasks/$($task.id)"
```

查看 SSE：

```powershell
curl.exe -N -b .\synapse.cookies "$base/v1/tasks/$($task.id)/events"
```

再次查询任务列表：

```powershell
curl.exe -s -b .\synapse.cookies "$base/v1/tasks?limit=10"
```

按状态过滤：

```powershell
curl.exe -s -b .\synapse.cookies "$base/v1/tasks?status=completed&limit=10"
```

## 取消任务

创建一个任务后立即取消：

```powershell
$cancelResp = curl.exe -s -b .\synapse.cookies `
  -H "Content-Type: application/json" `
  -d '{"reason":"manual validation"}' `
  "$base/v1/tasks/$($task.id)/cancel"

$cancelResp
```

说明：

| 场景 | 状态码 |
|---|---|
| queued/running/paused 首次取消 | 202 |
| 已取消任务再次取消 | 200 |
| completed/failed 取消 | 409 |

批量取消：

```powershell
curl.exe -s -b .\synapse.cookies `
  -H "Content-Type: application/json" `
  -d '{"task_ids":["task-id-1","task-id-2"],"reason":"batch validation"}' `
  "$base/v1/tasks/cancel"
```

## 审批恢复

触发审批通常需要高风险工具，例如 HTTP/浏览/code_exec，且未提供审批元数据。mock 回归中也覆盖该链路。

审批接口：

```powershell
curl.exe -s -b .\synapse.cookies `
  -H "Content-Type: application/json" `
  -d '{"reason":"manual approval","approved_tool_call":{"tool_name":"summarize_page","tool_input":"https://example.com","risk_level":"high","reason":"manual approval","resume_step_index":1}}' `
  "$base/v1/tasks/$($task.id)/approve"
```

仅 paused 任务可以审批，否则返回 `409`。

## 重放任务

```powershell
curl.exe -s -b .\synapse.cookies `
  -X POST `
  "$base/v1/tasks/$($task.id)/replay"
```

running 任务不可重放，其它状态会重置为 `queued` 并重新入队。

## 长期记忆

手工写入记忆：

```powershell
curl.exe -s -b .\synapse.cookies `
  -H "Content-Type: application/json" `
  -d '{"content":"Gateway retries are bounded and dead letters can be replayed.","summary":"bounded retry and replay","source_task_id":"manual","importance":0.8}' `
  "$base/v1/memories"
```

列出记忆：

```powershell
curl.exe -s -b .\synapse.cookies "$base/v1/memories?limit=5"
```

召回记忆：

```powershell
curl.exe -s -b .\synapse.cookies "$base/v1/memories/recall?query=gateway%20retries&limit=3"
```

删除记忆：

```powershell
$memories = curl.exe -s -b .\synapse.cookies "$base/v1/memories?limit=1" | ConvertFrom-Json
$memoryID = $memories.items[0].memory_id
curl.exe -s -b .\synapse.cookies -X DELETE "$base/v1/memories/$memoryID"
```

## 会话删除

聊天任务会在 metadata 中带 `conversation_id`。获取最近任务的会话 ID：

```powershell
$tasks = curl.exe -s -b .\synapse.cookies "$base/v1/tasks?limit=1" | ConvertFrom-Json
$conversationID = $tasks.items[0].metadata.conversation_id
$conversationID
```

删除会话：

```powershell
curl.exe -s -b .\synapse.cookies -X DELETE "$base/v1/conversations/$conversationID"
```

响应包含被删除的任务 ID。该操作只删除当前登录用户的会话。

## 死信列表

死信列表仅管理员可访问：

```powershell
curl.exe -s -b .\synapse.cookies "$base/v1/dead-letters?limit=20"
```

如果普通用户访问，预期返回 `403`。

## 回归评测

AI Engine mock 回归：

```powershell
Set-Location services/ai-engine-py
python -m app.benchmarks.regression
Set-Location ..\..
```

预期 `passed=true`，并展示工具、记忆、审批、浏览、失败恢复等覆盖率。

## 常见失败

| 现象 | 排查 |
|---|---|
| `/healthz` 返回 degraded | AI Engine 未启动或 `SYNAPSE_AI_ENGINE_ADDR` 配置错误 |
| 登录后仍 401 | curl 未带 `-b .\synapse.cookies` 或 Cookie 已过期 |
| 任务一直 queued | Worker 未运行或队列连接异常 |
| 任务 failed | 查看任务 `error` 和 SSE `failed` 事件 |
| SSE 无输出 | 确认 task id 正确，且使用 `curl.exe -N` |
| 记忆接口 502 | AI Engine Memory RPC 不可达 |
| 真实 provider 报错 | 检查 env 文件的 API Key、base_url、model 是否匹配 |
