# 功能：认证与权限

## 1. 功能目标

保障任务系统的用户隔离与运维权限边界，避免越权读写。

## 2. 相关实现

1. [services/gateway-go/internal/api/handlers_auth.go](../services/gateway-go/internal/api/handlers_auth.go)
2. [services/gateway-go/internal/domain/auth.go](../services/gateway-go/internal/domain/auth.go)
3. [services/gateway-go/internal/store/store.go](../services/gateway-go/internal/store/store.go)

## 3. 认证流程

1. 注册：创建 auth_users 记录（bcrypt 哈希密码）。
2. 登录：验证密码，创建 auth_sessions，写 Cookie。
3. 鉴权：从 Cookie 读取 token，查询有效会话。
4. 登出：删除会话并清理 Cookie。

## 4. 权限模型

1. user：仅能访问自己的任务与事件。
2. admin：可访问全局任务和运维能力（死信、运维台）。

## 5. API 权限规则

1. /v1/dead-letters：admin only。
2. /v1/tasks/{taskID} 与事件流：owner 或 admin。
3. 普通用户创建任务时 user_id 强制为当前登录用户名。

## 6. 安全基线

1. 密码存储采用 bcrypt。
2. Cookie 为 HttpOnly + SameSite=Lax。
3. 会话带过期时间并支持清理。

## 7. 风险与建议

1. 生产建议开启 HTTPS 并设置 Cookie Secure=true。
2. 建议增加 CSRF 保护与登录限流。
3. 建议新增审计日志，记录关键权限操作。
