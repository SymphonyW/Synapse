# 功能：任务生命周期与事件流

## 1. 功能目标

为任务提供可追踪、可恢复的状态机，并通过事件流驱动前端实时反馈。

## 2. 状态定义

状态定义位于 [services/gateway-go/internal/domain/task.go](../services/gateway-go/internal/domain/task.go)：

1. queued
2. running
3. completed
4. failed
5. canceled

## 3. 典型状态迁移

1. create：queued
2. worker 开始执行：running
3. 正常完成：completed
4. 异常失败：failed
5. 用户取消：canceled

## 4. 事件类型

常见事件：

1. info
2. started
3. token
4. cancel_requested
5. canceled
6. completed
7. failed
8. dead_lettered
9. replay_requested
10. terminal

## 5. SSE 机制

接口：GET /v1/tasks/{taskID}/events

机制：

1. 支持 last_event_id 增量拉取。
2. 每条事件包含 event_id。
3. 终态且无新事件时发送 terminal。

## 6. 取消语义

1. queued/running 首次取消：202。
2. 已取消再次取消：200（幂等）。
3. completed/failed 不可取消：409。

## 7. 前端消费要点

1. 维护 task 级 last_event_id 游标。
2. 去重 event_id，防止重连重复渲染。
3. token 事件直接拼接到输出缓冲。

## 8. 监控建议

1. 统计各状态数量和转化率。
2. 统计任务平均耗时与 P95。
3. 统计 token 事件吞吐与 SSE 活跃连接数。
