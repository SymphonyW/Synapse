# 功能：会话上下文构建

## 1. 功能目标

支持多轮对话场景，使模型请求携带近期历史语境，提升连续问答质量。

## 2. 实现位置

核心逻辑位于 [services/gateway-go/internal/api/handlers.go](../services/gateway-go/internal/api/handlers.go)。

关键元数据键：

1. conversation_id
2. user_message
3. client_view
4. model_prompt
5. model_messages_json

## 3. 构建流程

1. CreateTask 时识别会话任务。
2. 读取同 user_id + conversation_id 的历史任务。
3. 过滤仅 completed 轮次，避免失败文本污染上下文。
4. 从历史事件拼接 assistant 输出（优先 token）。
5. 生成 model_prompt 与 model_messages_json。
6. 将结构化 messages 写入 metadata，供 AI Runtime 优先使用。

## 4. 上下文窗口

1. 默认仅保留最近有限轮次（conversationContextTurnLimit）。
2. 事件加载有上限（conversationEventLoadLimit）。
3. 通过窗口机制防止 prompt 无界增长。

## 5. 价值

1. 前端无需自行维护完整 prompt 拼接逻辑。
2. 网关统一控制上下文策略，可审计、可演进。
3. 结构化 messages 兼容 OpenAI chat 语义。

## 6. 当前不足

1. 仍主要依赖“最近轮次”，缺少摘要压缩机制。
2. 超长会话可能仍导致成本上升。
3. 尚未区分系统指令、业务指令的多级优先级。

## 7. 演进建议

1. 增加历史摘要层（summary memory）。
2. 引入 token 预算控制与截断策略。
3. 支持会话级提示词模板与可配置上下文策略。
