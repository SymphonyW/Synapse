# 功能：重试、死信与重放

## 1. 功能目标

提高任务执行链路在外部不稳定场景下的韧性，避免单次失败直接导致任务不可恢复。

## 3. 重试机制

1. 最大重试次数：SYNAPSE_TASK_MAX_ATTEMPTS。
2. 重试间隔：SYNAPSE_TASK_RETRY_BACKOFF。
3. 重试过程中写入 info 事件，便于可观测。
4. 不可重试错误快速失败，减少无效消耗。

## 4. 不可重试错误示例

1. DeadlineExceeded
2. InvalidArgument
3. PermissionDenied
4. Unauthenticated
5. Unimplemented
6. FailedPrecondition

## 5. 死信机制

1. 最后一轮重试失败后写入 dead_letter_tasks。
2. 追加 failed 与 dead_lettered 事件。
3. 运维台可查询死信列表并发起重放。

## 6. 重放机制

1. ReplayTask 仅允许非 running 任务重放。
2. 状态重置为 queued。
3. 清理死信记录。
4. 追加 replay_requested 事件。
5. 重新入队等待执行。

## 7. 取消与死信关系

1. 任务进入取消流程时会清理死信记录。
2. finalizeCanceled 优先保留已有取消原因。
3. 取消不属于失败重试路径。

## 8. 运维建议

1. 对 dead_lettered 事件设置告警。
2. 定期分析死信原因分布（provider 失败、配置错误、权限错误）。
3. 对高频失败任务引入自动熔断或降级策略。
