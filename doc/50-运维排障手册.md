# 运维排障手册

## 1. 适用范围

用于本地开发、联调环境和测试环境中的常见问题快速定位。

## 2. 基础健康检查

1. 检查 Gateway：GET /healthz。
2. 检查 AI provider 显示值是否符合当前 env 预期。
3. 检查 Web 是否可访问并成功登录。

## 3. 常见故障与处理

### 3.1 任务长时间 running 无输出

排查顺序：

1. 确认 AI Engine 容器环境变量是否正确注入。
2. 检查 SYNAPSE_OPENAI_BASE_URL 与 provider 是否匹配。
3. 检查 API key 是否有效。
4. 查看任务事件流是否有 failed 或重试信息。

### 3.2 批量取消后部分任务仍在运行

1. 查看批量取消返回 failed 明细。
2. 对失败 ID 再次发起单任务取消。
3. 若任务已终态，返回 409 属于预期。

### 3.3 死信任务持续增长

1. 统计失败原因并按类型聚类。
2. 若为配额或限流，调整重试与流量策略。
3. 若为鉴权问题，优先修复密钥配置。

### 3.4 前端事件流断开

1. 检查浏览器网络连接与代理设置。
2. 检查 Gateway 日志中 SSE 接口状态码。
3. 前端会自动重建游标，建议切换任务触发重连。

### 3.5 任务进入 paused 长时间未恢复

1. 在事件流中确认是否出现 approval_required 或 paused 事件。
2. 使用运维台“审批并恢复”按钮，或调用 POST /v1/tasks/{taskID}/approve。
3. 优先核对 approved_tool_call 是否匹配 metadata 中的 agent_required_tool、agent_required_tool_input、agent_required_tool_risk_level。
4. 旧任务再核对 approved_tools 是否覆盖任务要求的 tool（metadata.agent_required_tool）。
5. 若审批后仍失败，检查 AI Engine 工具策略与 allowlist 配置。

### 3.6 长期记忆接口失败

1. Gateway 返回 502 时，优先检查 AI Engine gRPC 是否可达。
2. Gateway 返回 memory backend unavailable 时，检查 SYNAPSE_AGENT_MEMORY_FILE 是否为空或不可写。
3. Docker Compose 默认将 AI Engine 记忆文件挂载到 ai_engine_data volume。
4. 使用 GET /v1/memories/recall?query=... 验证召回是否有命中。

### 3.7 Web 能登录但页面无任务更新

1. 确认 Vite 代理仍指向 http://127.0.0.1:8080。
2. 检查浏览器请求是否带 Cookie。
3. 直接用 [05-接口验证手册](05-接口验证手册.md) 中的 curl 命令验证 Gateway。

## 4. 建议排障命令（PowerShell）

```powershell
# 启动全栈（智谱 + 镜像）
.\scripts\dev.ps1 -Task up-zhipu-mirror

# 查看 Gateway 单元测试
Set-Location services/gateway-go
go test ./...

# 构建前端
Set-Location ..\..\apps\web
npm run build

# AI Engine 单元测试与回归
Set-Location ..\..\services\ai-engine-py
python -m unittest discover -s tests -p "test_*.py"
python -m app.benchmarks.regression
```

## 5. 运行观察项

1. Gateway 请求日志中的状态码与耗时。
2. dead_letter_tasks 数量变化。
3. tasks 中 running 持续时间异常任务。
4. SSE token 事件吞吐是否突降。
5. paused 任务数量与平均审批恢复耗时。

## 6. 值班建议

1. 先判断是配置问题、网络问题还是业务输入问题。
2. 保留失败任务 ID 与 trace_id，便于跨层追踪。
3. 对重复问题沉淀到本手册，缩短后续恢复时间。
