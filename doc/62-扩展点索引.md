# 扩展点索引

本文按模块列出当前仓库最适合继续扩展的位置、预留痕迹和边界风险。扩展时应优先复用既有抽象，避免把同一职责散落到多个运行时。

## 1. Gateway API

| 扩展点 | 文件路径 | 适合扩展什么 | 预留痕迹 | 边界风险 |
|---|---|---|---|---|
| `Handler` | [services/gateway-go/internal/api/handlers.go](../services/gateway-go/internal/api/handlers.go) | 任务查询、审批、取消、重放、会话操作的 HTTP 编排 | `NewHandler` 通过接口注入 store、agentClient、taskQueue、taskCanceler | 不要在 Handler 内直接执行 Agent 或模型逻辑；应继续交给 Worker/AI Engine |
| `CreateTask` | [services/gateway-go/internal/api/handlers.go](../services/gateway-go/internal/api/handlers.go) | 新任务 metadata、会话上下文、任务入口校验 | 已有 `conversation_id`、`model_messages_json`、`auth_user_role` 注入 | 不要信任客户端传入的 `user_id`、`auth_*`、`model_messages_json` |
| `ApproveTask` | [services/gateway-go/internal/api/handlers.go](../services/gateway-go/internal/api/handlers.go) | 审批恢复表单、审批审计、审批策略扩展 | 已支持 `approved_tool_call` 和旧版 `approved_tools` | 粗暴设置 `approval_granted=true` 会削弱精确审批，应保留 tool call 匹配 |
| `StreamTaskEvents` | [services/gateway-go/internal/api/handlers.go](../services/gateway-go/internal/api/handlers.go) | SSE 心跳、事件过滤、Trace 只读查询 | 已支持 `last_event_id` 和 terminal 帧 | 不要绕过 TaskStore 直接读 Worker 内存状态，否则断线续传会失效 |
| `handlers_memory.go` | [services/gateway-go/internal/api/handlers_memory.go](../services/gateway-go/internal/api/handlers_memory.go) | 记忆管理 UI、管理员 user_id 查询、召回调试 | 已有 `ListMemories`、`WriteMemory`、`RecallMemory`、`DeleteMemory` | 不要把记忆存储迁到 Gateway；Gateway 当前边界是权限和 gRPC 转发 |
| `NewRouter` | [services/gateway-go/internal/api/router.go](../services/gateway-go/internal/api/router.go) | 新增 HTTP API、访问日志增强 | 使用 Go 1.22 pattern mux 注册路由 | 新路由要同步文档和权限规则，避免出现未认证管理接口 |

## 2. Gateway Worker、存储与队列

| 扩展点 | 文件路径 | 适合扩展什么 | 预留痕迹 | 边界风险 |
|---|---|---|---|---|
| `TaskProcessor` | [services/gateway-go/internal/worker/processor.go](../services/gateway-go/internal/worker/processor.go) | 并发 worker、trace_id 贯穿、重试策略、暂停统计 | `ProcessorOptions` 已集中控制超时、最大尝试、退避 | 不要在 Worker 中解释工具业务结果；Worker 只应识别少数状态驱动事件 |
| `parseApprovalRequiredInfo` | [services/gateway-go/internal/worker/processor.go](../services/gateway-go/internal/worker/processor.go) | 审批上下文字段扩展 | 已提取 tool、tool_input、risk_level、resume_step_index、reason | 修改 info JSON 结构时必须保持旧字段兼容，否则历史暂停任务无法恢复 |
| `TaskStore` | [services/gateway-go/internal/store/store.go](../services/gateway-go/internal/store/store.go) | 新存储后端、分页查询、Trace 查询、Replay Diff 数据读取 | 接口已覆盖任务、事件、死信、认证会话 | 大幅扩接口前应先确认 InMemory/Postgres 两个实现能同步演进 |
| `PostgresStore.ensureSchema` | [services/gateway-go/internal/store/postgres.go](../services/gateway-go/internal/store/postgres.go) | migration 迁移、索引优化 | 当前自动 `CREATE TABLE IF NOT EXISTS` 降低启动门槛 | 直接改表结构会影响已有本地数据，应引入版本化 migration |
| `TaskQueue` | [services/gateway-go/internal/queue/queue.go](../services/gateway-go/internal/queue/queue.go) | Redis Streams、外部 MQ、ack/reclaim | 抽象只有 `Enqueue/Dequeue/Close`，便于替换 | 如果需要 ack，需要扩接口并同步改 Worker，不应在 Redis 实现里私藏语义 |
| `RedisQueue` | [services/gateway-go/internal/queue/redis.go](../services/gateway-go/internal/queue/redis.go) | 队列可靠性升级 | 当前使用 `LPUSH` + `BRPOP` | 继续用 List 无法解决 Worker 崩溃后的 reclaim |

| `AgentRuntime.__init__` | [services/ai-engine-py/app/runtime.py](../services/ai-engine-py/app/runtime.py) | 新 provider、新工具 provider、策略默认值、审计配置 | `agent_tool_providers` 参数已支持外部 provider 注入 | 不要让 provider 直接执行前绕过 `ToolPolicy` 和 `ToolAuditLogger` |
| `run_task` | [services/ai-engine-py/app/runtime.py](../services/ai-engine-py/app/runtime.py) | Agent loop 阶段、记忆召回、最终合成、memory_write、evaluate | 已输出标准 `synapse.agent.info.v1` 事件 | 新阶段应保持 info envelope 结构，避免 Web/Benchmark 解析断裂 |
| `_planner_select_tool` / `_select_tool` | [services/ai-engine-py/app/runtime.py](../services/ai-engine-py/app/runtime.py) | 模型工具选择、策略规划器 | 代码注释说明 OpenAI provider 目前仍回退确定性策略 | 直接让模型自由调用工具会绕过审批预期，必须先产出 `PlannerDecision` |
| `_executor_run_decision_events` | [services/ai-engine-py/app/runtime.py](../services/ai-engine-py/app/runtime.py) | 工具执行、审批暂停、审计增强 | 已统一处理 role allow、approval_required、approved_tool_call | 不要在具体工具里重复审批判断；审批应保留在 Runtime executor 层 |
| `_execute_tool` | [services/ai-engine-py/app/runtime.py](../services/ai-engine-py/app/runtime.py) | ToolCall 参数适配、插件工具执行 | `_build_tool_call_arguments` 已为内置和单字段插件做兼容 | 不要让插件直接接收原始 prompt 执行副作用，应走 `ToolCall`/`ToolContext` |
| `_run_openai` 相关方法 | [services/ai-engine-py/app/runtime.py](../services/ai-engine-py/app/runtime.py) | 真实模型路由、续写策略、fallback、成本统计 | 已支持 OpenAI-compatible stream、fallback、长文本续写 | 加 provider 路由时要保持 `model_messages_json` 输入优先级 |

## 5. 长期记忆
